#速度与行动序

来源：布局与行动.txt — 按索引0→8遍历，同索引双方存在时速度对比，高速先动

#速度属性设计（建议补充至锚点设计）

当前已定义属性：攻击,生命,防御,暴击,格挡
建议新增：速度

建议锚点：
    初始速度=100
    成长速度=10

速度属性同样适用公式：速度=100 + 10*等级*1.023293^等级

等级    速度
1       100 + 10*1*1.023293 = 110.2
10      100 + 10*10*1.023293^10 = 225.9
30      100 + 10*30*1.023293^30 = 698.6
60      100 + 10*60*1.023293^60 = 2488.7
100     100 + 10*100*1.023293^100 = 10100.0

#行动序遍历规则

每回合从索引0开始，依次遍历0→1→2→3→4→5→6→7→8：

索引    判断                              行动
0       双方0号位均有角色                  速度高者先行动，速度低者后行动
0       仅我方0号位有角色                  我方直接行动
0       仅敌方0号位有角色                  敌方直接行动
0       双方0号位均无角色                  跳过，继续遍历索引1

索引1~8同理。一轮遍历完毕即一回合结束，回合数+1，重新从索引0开始。

#位置与先手权

因遍历顺序固定0→8，位置本身决定行动优先级：
    索引0 > 索引1 > 索引2 > ... > 索引8

索引0拥有绝对先手权（全场最先行动），索引8最后行动。
前排0/1/2整体优先于中排3/4/5，中排优先于后排6/7/8。

战术意义：
- 将核心输出放在索引0可确保先手爆发
- 将治疗放在索引0可抢先治疗
- 将控制角色放在索引0可先手控制敌方核心
- 坦克放前排天然获得先手权

#速度差异的影响

速度只在同索引对比时生效，不同索引之间速度不影响顺序。

示例（索引0，我方速度110 vs 敌方速度120）：
    敌方速度120 > 我方速度110 → 敌方先行，我方后行

速度优势在同索引对抗中提供先手，但不跨越索引顺序。

#同速处理

当同索引双方速度相等时：
    建议：随机判定先手，或固定我方优先（PVE） / 随机（PVP）

#装备对速度的影响

速度属性同样可受装备加成（建议补充至装备设定.md）：

部位      附加属性      权重参考
鞋子/防具  速度          1/4防御+1/4生命之外可附加速度
饰品      速度          1/4攻击+1/4生命之外可附加速度

具体数值遵循装备设定.md公式：速度资质* (n+5) * 1.023293^(n+5) * 波动 * 权重

#先手价值量化

先手优势：先行动方可先输出伤害/治疗/控制，减少对方下一击的有效HP。

在14回合普攻TTK中，先手约可获得1回合的行动优势，相当于7%的TTK优势。
在4回合技能TTK中，先手优势更显著（25%回合优势）。

速度因此是重要的PvP竞争属性，建议速度差距设计在±30%以内以避免绝对碾压。

#全局技能判定插入

全局技能在每次行动后检查，即：
    索引遍历 → 角色行动 → 全局技能检查(遍历双方所有角色) → 可能插入技能

插入的技能不占用行动序，不跳过当前索引的后续行动。
同一索引内可能因全局技能连锁触发多次插入。
